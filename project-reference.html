Stack Architecture &
Implementation Reference

divergence-MCP is an architecture-first build demonstrating governed AI infrastructure through deliberate, incremental construction. Each layer is added with explicit reasoning, audit intent, and documented governance rationale.

Project Overview

divergence-MCP

divergence-MCP establishes the foundational runtime, tool, and infrastructure architecture for governed AI agent operations. It is not a demonstration environment — it is the production-intent scaffold from which sector-specific deployments are derived.

The project name reflects a deliberate divergence from unstructured AI deployment patterns toward a model where governance controls, cost instrumentation, and audit traceability are structural properties rather than applied overlays.

Architecture Layers

Four-Layer Stack Reference

"The distinction between experimental AI and governed AI is not the sophistication of the model — it is the presence of structural controls around model execution. divergence-MCP builds those controls from the ground up."

Runtime

Execution Environment

Layer 01

Node.js ESM provides the runtime contract. ESM boundaries enforce clean module isolation, preventing implicit state leakage between agent sessions. Model versioning is explicit — no aliases permitted.

API key management via environment variables validated. No credentials embedded in agent logic.

Tool Layer

MCP Tool Definitions

Layer 02

Three core tools deployed via MCP SDK. Each tool is defined with explicit JSON Schema, enabling validation before execution. Schema validation at the protocol layer prevents malformed calls reaching infrastructure.

Tools are versioned independently of agent logic, enabling governance updates without workflow redeployment.

Infrastructure

Persistence & Integration

Layer 03

PostgreSQL serves as the primary audit and cost logging backend. Schema is append-only — records are never updated, only appended. This tamper-evident design is a regulatory requirement in most target sectors.

AWS Cost Explorer API integration extends visibility to include underlying infrastructure costs attributable to AI workloads.

Orchestration

Workflow & Composition

Layer 04

n8n provides HTTP and SSE transport for agent orchestration. Governance logic — escalation paths, approval gates — can be updated independently of model logic. This separation is architecturally critical.

Cross-server MCP composition enables specialised servers to compose into single workflows with each server maintaining its own tool namespace.
